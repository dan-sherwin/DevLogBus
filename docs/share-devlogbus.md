## One Line

DevLogBus is a local-first structured log bus that brings backend, CLI, Linux
journal, and browser debugging records into one live development stream.

## Short Blurb

DevLogBus is local developer tooling for seeing cause and effect across services,
CLIs, Linux journals, and browser tabs without juggling five terminals and a
browser console. It includes a daemon, CLI, terminal UI, browser UI, Chrome
Browser Tap extension, package-manager installs, and SDKs for Go, C, .NET/C#,
Rust, Java/Kotlin, Node/TypeScript, and Python.

## Longer Blurb

DevLogBus is a local-first structured log bus for development and active
troubleshooting. It gives backend services, command-line tools, Linux
`journald`, browser console/runtime/network events, and SDK-published records a
single stream with browser and terminal viewers. It is not a production
observability platform; it is a workstation and trusted-network tool for the
part of debugging where you need signal immediately and do not want to build a
whole logging stack just to understand what happened.
